Instrucciones de Configuración
Para configurar y probar el sistema en su máquina local, siga los siguientes pasos:
Preparación del Entorno
  1. Asegúrese de tener Python 3.8+ instalado en su sistema
  2. Instale PostgreSQL 13+ y cree una base de datos llamada "gestor_evidencias"
  3. Clone el repositorio en su máquina local
git clone https://github.com/ministerio-publico/gestor_evidencias_mp.git
cd gestor_evidencias_mp
Configuración del Entorno Virtual
  1. Cree un entorno virtual para el proyecto
  2. Active el entorno virtual
  3. Instale las dependencias requeridas
# Crear entorno virtual
python -m venv venv

# Activar entorno virtual (Windows)
venv\Scripts\activate

# Activar entorno virtual (Linux/Mac)
source venv/bin/activate

# Instalar dependencias
pip install -r requirements.txt
Configuración del Proyecto
  1. Cree un archivo .env basado en el archivo env.example
  2. Configure las variables de entorno de acuerdo a su entorno local
# Windows
copy env.example .env

# Linux/Mac
cp env.example .env

# Edite el archivo .env con su configuración de base de datos
Preparación de la Base de Datos
  1. Ejecute las migraciones para crear las tablas en la base de datos
  2. Cree un superusuario para acceder al sistema
# Aplicar migraciones
python manage.py migrate

# Crear superusuario
python manage.py createsuperuser
Ejecución del Servidor de Desarrollo
  1. Inicie el servidor de desarrollo de Django
  2. Acceda a la aplicación a través de su navegador web
# Iniciar servidor de desarrollo
python manage.py runserver

# Acceda a la aplicación en: http://localhost:8000/
Ejecución de Pruebas
  1. Ejecute las pruebas unitarias para verificar que todo funciona correctamente
# Ejecutar todas las pruebas
pytest

# Ejecutar pruebas específicas
pytest tests/test_auth.py
Uso con Docker

Alternativamente, puede usar Docker y Docker Compose para ejecutar todo el sistema sin necesidad de instalar PostgreSQL localmente:

# Copiar archivo de variables de entorno
copy env.example .env  # (Windows)
cp env.example .env    # (Linux/Mac)

# Editar archivo .env según sea necesario

# Construir y levantar los contenedores
docker-compose up -d

# Crear superusuario
docker-compose exec web python manage.py createsuperuser

# Acceda a la aplicación en: http://localhost:8000/
